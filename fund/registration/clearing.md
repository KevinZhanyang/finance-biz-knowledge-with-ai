# 清算交收

## 清算 vs 交收

|  | 清算（Clearing） | 交收（Settlement） |
|--|-----------------|-------------------|
| 做什么 | **算账**：计算各方应收应付金额 | **划钱**：实际完成资金转移 |
| 时间 | T 日日终 | T+1 日（或更晚） |
| 产出 | 清算文件（各渠道应收应付汇总） | 资金到账确认 |
| 类比 | 饭后算账，谁该付多少 | 实际掏钱转账 |

> 先清算，后交收。清算是逻辑计算，交收是物理执行。

## 为什么需要清算

一家基金公司可能对接上百家代销机构，每天有成千上万笔申购赎回。如果逐笔划款，效率极低。清算的作用就是**汇总轧差**：

```
某代销机构 T 日交易汇总：
  申购 500 笔，总金额 2,000 万元  （代销机构应付给基金）
  赎回 300 笔，总金额 1,200 万元  （基金应付给代销机构）

净额清算：
  净应付 = 2,000 万 - 1,200 万 = 800 万元
  → 代销机构只需向基金划付 800 万元（而非分别划付 2,000 万和 1,200 万）
```

## 清算层级

基金清算涉及多个层级，从下到上逐层汇总：

```
第一层：交易级清算（TA 系统）
  逐笔计算每笔交易的确认金额、手续费
      ↓
第二层：渠道级清算（TA 系统）
  按代销机构汇总，计算每个渠道的净应收/应付
      ↓
第三层：基金级清算（TA → 托管银行）
  按基金汇总，生成清算指令
      ↓
第四层：资金划付（托管银行执行）
  通过银行间支付系统完成实际资金转移
```

## 场外基金清算流程

场外基金（通过银行、第三方平台等代销渠道销售的基金）的清算是最常见的场景。

### 完整流程

```
T 日
─────────────────────────────────────────
15:00    交易截止
  ↓
17:00-20:00  净值确认
  ↓
20:00-24:00  TA 日终跑批
             ├── 逐笔交易确认（份额/金额计算）
             ├── 按渠道汇总净应收应付
             └── 生成清算文件

T+1 日
─────────────────────────────────────────
08:00-09:00  TA 将清算文件发送给托管银行
  ↓
09:00-10:00  托管银行复核清算数据
  ↓
10:00-16:00  资金划付
             ├── 净申购 > 净赎回：代销机构 → 基金托管账户
             └── 净赎回 > 净申购：基金托管账户 → 代销机构
  ↓
16:00-17:00  资金到账确认、对账
```

### 清算数据内容

TA 生成的清算文件通常包含：

| 字段 | 说明 |
|------|------|
| 基金代码 | 哪只基金 |
| 代销机构代码 | 哪个渠道 |
| 申购确认金额 | 该渠道当日申购总金额 |
| 赎回确认金额 | 该渠道当日赎回总金额 |
| 申购手续费 | 该渠道当日申购手续费总额 |
| 赎回手续费 | 该渠道当日赎回手续费总额 |
| 净清算金额 | 申购金额 - 赎回金额（正数=代销应付，负数=基金应付） |
| 清算日期 | T 日 |
| 交收日期 | T+1 日 |

## 场内基金清算流程

场内基金（ETF、LOF 场内交易部分）通过**中国结算**（中登公司）进行集中清算。

### 与场外清算的区别

|  | 场外清算 | 场内清算 |
|--|---------|---------|
| 清算主体 | 基金公司 TA + 托管银行 | 中国结算（中登） |
| 清算方式 | 基金公司与各代销机构逐一清算 | 中登统一多边净额清算 |
| 交收周期 | T+1 | T+1 |
| 资金通道 | 银行间支付系统 | 中登结算系统 |
| 对手方 | 基金公司 ↔ 各代销机构 | 中登作为中央对手方（CCP） |

### 中央对手方机制

```
场外清算（双边）：
  代销A ←→ 基金公司
  代销B ←→ 基金公司
  代销C ←→ 基金公司

场内清算（中央对手方）：
  券商A ←→ 中登 ←→ 券商B
               ↕
             券商C

  中登介入每笔交易，成为买方的卖方、卖方的买方
  各券商只需与中登清算，不需要与对手方逐一清算
```

## 资金流向详解

### 申购资金流

```
投资者银行账户
    ↓ ① 扣款
代销机构备付金账户（归集资金）
    ↓ ② T+1 划付
基金托管账户（托管银行）
    ↓ ③ 基金经理投资
证券市场 / 银行间市场
```

- ① 投资者提交申购时，资金从银行卡扣至代销机构的备付金账户
- ② T+1 日清算交收时，代销机构将净申购款划至基金托管账户
- ③ 基金经理使用托管账户资金进行投资

### 赎回资金流

```
证券市场 / 银行间市场
    ↓ ③ 基金经理卖出变现（如需）
基金托管账户（托管银行）
    ↓ ② T+1 划付
代销机构备付金账户
    ↓ ① T+1~T+7 发放
投资者银行账户
```

- 赎回方向相反：托管账户 → 代销机构 → 投资者
- 代销机构收到资金后才向投资者发放（不同基金到账时效不同，见[申购赎回流程](../sales/subscription.md)）

### 备付金账户

代销机构通常设有**备付金账户**（也叫清算备付金账户），用于：
- 归集投资者的申购资金
- 暂存待发放的赎回资金
- 与基金公司/托管银行之间的资金划转中转站

> 备付金账户的资金属于投资者，代销机构不得挪用。监管要求实行严格的备付金管理。

## 费用清算

清算不仅涉及申购赎回资金，还包括各类费用的结算：

### 管理费和托管费

```
每日计提：
  日管理费 = 前一日基金净资产 × 年管理费率 / 当年天数
  日托管费 = 前一日基金净资产 × 年托管费率 / 当年天数

按月支付：
  每月末从基金资产中扣除当月累计的管理费和托管费
  管理费 → 支付给基金公司
  托管费 → 支付给托管银行
```

### 尾随佣金（客户维护费）

```
按季/月结算：
  尾随佣金 = 该渠道日均保有量 × 年费率 / 当年天数 × 天数

基金公司从已收取的管理费中支付给代销机构
```

### 手续费分成

```
申购手续费 → 大部分归代销机构（通常代销机构拿 70%-100%）
赎回手续费 → 部分归基金资产（持有 < 7天的赎回费，≥75% 归基金资产）
              部分归代销机构
```

## 清算差错处理

### 常见差错类型

| 差错类型 | 原因 | 处理方式 |
|---------|------|---------|
| 份额差异 | TA 确认数据与代销机构记录不一致 | 以 TA 数据为准，代销机构调账 |
| 资金差异 | 清算金额与实际划付金额不一致 | 查明原因后补划或退回 |
| 漏单 | 代销机构申请文件未送达 TA | 补报后在后续交易日处理 |
| 净值错误 | 估值错误导致份额/金额计算有误 | 基金公司公告净值更正，对受影响交易重新确认 |

### 差错处理原则

- **以 TA 为准**：份额数据以 [TA 系统](./ta-system.md)登记为权威
- **时效要求**：差错须在发现后规定时间内处理（通常 T+2 内）
- **损失承担**：因基金公司或 TA 系统错误导致的损失由基金公司承担

## 清算时效汇总

| 基金类型 | 清算周期 | 资金交收 | 投资者到账 |
|---------|---------|---------|-----------|
| 货币基金 | T 日 | T+0 或 T+1 | T+0（快赎限1万）/ T+1 |
| 债券基金 | T 日 | T+1 | T+2 ~ T+3 |
| 股票/混合基金 | T 日 | T+1 | T+3 ~ T+5 |
| QDII 基金 | T 日 | T+2 ~ T+3 | T+4 ~ T+10 |
| ETF（场内） | T 日 | T+1 | T+1（资金可用） |

> QDII 基金因涉及跨境资金划转和外汇结算，清算周期明显更长。

## 清算在全链路中的位置

```
投资者下单 → 销售机构受理 → TA交易确认 → 清算计算 → 资金交收 → 对账
                                          ~~~~~~~~
                                          本文内容

上游依赖：TA 确认结果（没有确认就无法清算）
下游驱动：资金划付（没有清算就不知道划多少钱）
```

清算是连接**份额世界**（TA 确认了多少份额）和**资金世界**（实际要划多少钱）的桥梁。
