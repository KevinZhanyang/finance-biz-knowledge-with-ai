# 认购/申购/赎回流程

## 认购 vs 申购

|  | 认购 | 申购 |
|--|------|------|
| 阶段 | 募集期（基金尚未成立） | 基金成立后的开放期 |
| 价格 | 固定面值 1.000 元 | 当日单位净值（未知价） |
| 费用 | 认购费（通常 ≤1.2%） | 申购费（按金额分档） |
| 确认 | 募集结束后统一确认 | T 日申请，T+1 确认 |
| 份额到账 | 基金成立后 + 建仓期结束 | T+1 可查 |

> 简单理解：认购是"打新"，申购是"日常买入"。

## 未知价法

**未知价法**是中国开放式基金交易的基本定价原则：

- 投资者提交申购/赎回申请时，**不知道**成交价格
- 成交价格 = T 日收市后计算出的**基金单位净值**
- 因此所有 T 日 15:00 前的申请，使用的是同一个净值

```
投资者 T 日 10:30 申购     ──┐
投资者 T 日 14:50 申购     ──┤──→ 均以 T 日净值确认
投资者 T 日 14:59 申购     ──┘

投资者 T 日 15:01 申购     ──→ 算 T+1 日申请，以 T+1 日净值确认
```

### 为什么用未知价法

- **公平性**：所有投资者使用相同净值，防止利用已知价格套利
- **防止抢跑**：如果用已知价（如前一日净值），当日市场大涨时会引发大量套利申购

### 15:00 分界线

- **15:00 前**提交 → T 日申请 → T 日净值
- **15:00 后**提交 → T+1 日申请 → T+1 日净值
- 这个时间点是**严格分界**，精确到秒
- 不同销售渠道可能有提前截止时间（如银行可能 14:30 截止内部处理）

## 申购流程

### 完整链路

```
投资者下单（选基金、输金额）
    ↓
销售机构受理（校验账户、适当性、限额）
    ↓
资金冻结（从投资者账户冻结申购金额）
    ↓
销售机构汇总当日申请 → 发送给基金公司 TA 系统
    ↓
15:00 收市 → 基金估值 → 净值计算 → 净值发布
    ↓
TA 日终跑批：按净值计算份额
    确认份额 = (申购金额 - 申购费) / T日单位净值
    ↓
TA 生成确认数据 → 回传给销售机构
    ↓
销售机构更新投资者账户（份额到账）
    ↓
资金清算：销售机构将申购资金划付至基金托管账户
```

### 时效

| 节点 | 时间 |
|------|------|
| 申请受理 | T 日（15:00前） |
| 净值确认 | T 日收市后 |
| 份额确认 | T 日日终 |
| 份额可查 | T+1 日 |
| 资金划付 | T+1 日 |

### 申购金额计算示例

```
申购金额：10,000 元
申购费率：1.5%（金额 < 100万）
T日单位净值：1.2350 元

净申购金额 = 10,000 / (1 + 1.5%) = 9,852.22 元
申购费     = 10,000 - 9,852.22 = 147.78 元
确认份额   = 9,852.22 / 1.2350 = 7,977.91 份
```

> 注意：申购费是"内扣法"，从申购金额中扣除，而非额外收取。

## 赎回流程

### 完整链路

```
投资者下单（选基金、输份额）
    ↓
销售机构受理（校验份额余额、最低持有份额）
    ↓
份额冻结（冻结拟赎回份额，防止重复赎回）
    ↓
销售机构汇总当日申请 → 发送给基金公司 TA 系统
    ↓
15:00 收市 → 基金估值 → 净值计算 → 净值发布
    ↓
TA 日终跑批：按净值计算赎回金额
    赎回金额 = 赎回份额 × T日单位净值 - 赎回费
    ↓
TA 生成确认数据 → 回传给销售机构
    ↓
基金托管账户划付赎回资金 → 销售机构 → 投资者
```

### 时效

| 节点 | 时间 |
|------|------|
| 申请受理 | T 日 |
| 净值确认 | T 日收市后 |
| 份额扣减 | T 日日终 |
| 资金到账 | 不同基金类型差异大（见下表） |

### 赎回到账时间

| 基金类型 | 资金到账 | 说明 |
|---------|---------|------|
| 货币基金 | T+0 或 T+1 | 快速赎回通常 T+0（单日限1万） |
| 债券基金 | T+2 ~ T+3 | |
| 股票/混合基金 | T+3 ~ T+5 | |
| QDII基金 | T+4 ~ T+10 | 涉及跨境结算，耗时最长 |

> 以上为典型时效，不同基金合同约定可能不同。

### 赎回金额计算示例

```
赎回份额：5,000 份
持有天数：200 天
赎回费率：0.5%（持有 < 1年）
T日单位净值：1.3500 元

赎回总额 = 5,000 × 1.3500 = 6,750.00 元
赎回费   = 6,750.00 × 0.5% = 33.75 元
实际到账 = 6,750.00 - 33.75 = 6,716.25 元
```

## 巨额赎回

### 定义

单个开放日，基金的**净赎回申请**（赎回总份额 - 申购总份额）超过前一日基金总份额的 **10%**，即为巨额赎回。

### 为什么会发生

- 市场恐慌导致集中赎回
- 某个大客户（机构）集中赎回
- 基金业绩持续不佳引发的"赎回潮"

### 处理方式

基金公司有两种选择：

#### 方式一：全额接受

- 接受当日全部赎回申请，正常确认
- 风险：基金可能需要被动卖出持仓，冲击投资组合
- 通常在基金流动性充裕时采用

#### 方式二：延期办理（部分确认）

```
当日赎回申请：800万份
前一日总份额：5000万份
净赎回比例：800万/5000万 = 16% > 10%  → 触发巨额赎回

处理方式：
  当日确认比例 = 10% × 5000万 / 800万 = 62.5%
  当日确认份额 = 800万 × 62.5% = 500万份
  延期份额     = 800万 - 500万 = 300万份 → 转入下一开放日处理
```

- 按比例确认当日可处理的份额
- 未确认部分自动转入下一个开放日继续赎回
- 延期部分按**后续确认日的净值**计算（非原申请日净值）
- 投资者也可在得知延期后选择撤销未确认部分

### 连续巨额赎回

如果连续多个开放日都触发巨额赎回，基金公司可以：
- 暂停接受赎回申请（最长不超过 20 个工作日）
- 启动基金清盘程序

## 其他交易类型

### 基金转换

将持有的 A 基金份额直接转换为同一基金公司的 B 基金份额，无需先赎回再申购。

```
转出基金净值 × 转出份额 - 转换费 = 可转入金额
可转入金额 / 转入基金净值 = 转入份额
```

- **优点**：节省时间（T+1确认 vs 赎回T+3再申购T+1）、可能节省费用
- **限制**：只能在同一基金公司旗下产品间转换

### 定期定额投资（定投）

- 投资者设定固定金额、固定周期（如每月500元）
- 销售机构按约定日期自动扣款申购
- 本质上是多次申购的自动化执行
- **优势**：摊平成本、纪律性投资、降低择时风险

### 分红

基金将已实现收益分配给持有人，两种方式：

| 方式 | 说明 |
|------|------|
| 现金分红 | 直接发放现金到投资者账户 |
| 红利再投资 | 分红金额自动按除息日净值再买入份额（通常免申购费） |

> 分红后基金净值会相应下降（除息），投资者总资产不变。

## 特殊场景

### 大额申购限制

基金公司可能限制单笔或单日申购上限，常见原因：
- 基金规模已过大，影响投资操作
- 避免大额资金摊薄存量持有人收益（"大额资金冲击"）
- 基金经理策略需要控制规模

### 最低持有份额

赎回时需保留最低份额（如 1000 份），否则须全部赎回。

### 封闭期限制

- 新基金成立后的建仓期（通常 ≤3个月）不接受赎回
- 定开基金在封闭期内不接受申购赎回
- 持有期基金：每笔份额有最短持有期限制

```
                         校验适当性
                         记录申请
                              ↓
15:00                    截止受理
                         汇总当日申请 ───→  接收申请文件
                                                              收市估值
17:00-19:00                                                   ←→ 净值复核
                                           接收确认净值
20:00-22:00                                日终跑批
                                           计算份额
                                           登记份额
                                           生成确认文件
                         ←───────────────  回传确认数据
─────────────────────────────────────────────────────────────────────────
T+1 上午    查询份额到账   更新客户账户       清算文件 ──→    资金划付
```
